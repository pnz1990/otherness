# 10: Multi-Agent Simulation — Empirical Grounding for the Self-Generating Vision Model

> Status: Active | Created: 2026-04-17
> Applies to: otherness self-improvement tooling

---

## What this does

A runnable mathematical simulation of the otherness multi-agent system. Multiple
parallel agents work on the same project simultaneously, competing for queue items,
generating vision proposals, producing Type B failures, accumulating anomalies.

The simulation measures whether the self-generating vision loop actually compounds
or converges to a local maximum, how parallel agent count affects this, and when
human inflection points are structurally necessary.

This is empirical grounding for the hypotheses generated by adversarial reasoning.
Hypotheses need independent evidence. The simulation provides evidence the narrative
cannot — it runs forward without steering, produces measurable outputs, and can be
rerun with different parameters to isolate variables.

**The core question:** Does adding agents accelerate compounding, or does monoculture
pressure cancel the gains? There is an optimal N — the simulation finds it.

## The boldness model

Boldness is the single most important state variable. It measures how far the
system's current vision proposals are from its current capabilities. Low boldness
= local refinement. High boldness = genuine leaps.

Boldness is governed by three simultaneous forces:

### Force 1 — Natural decay (conservative pull)

After each successful ship, boldness decays. The system built something; it wants
to protect it. Each ✅ transition applies a decay factor:

```
boldness(t+1) = boldness(t) × decay_rate
decay_rate ∈ (0.85, 0.95)  # tunable parameter
```

This models the system getting conservative after shipping. Without countervailing
forces, boldness converges to zero — pure local refinement, no leaps.

### Force 2 — Skill growth (gradual recovery)

Each item the system ships adds to its skill set. More skills expand the visible
possibility space — the system can see further because it has proved more. Skill
growth applies an additive lift to boldness each cycle:

```
skill_lift(t) = skill_count(t) × skill_boldness_coefficient
boldness(t+1) += skill_lift(t)
skill_boldness_coefficient ∈ (0.01, 0.05)  # tunable
```

Force 2 counteracts Force 1. In equilibrium (no Type B failures), boldness
oscillates around a stable point determined by the ratio of decay to skill growth.
That stable point is a local maximum.

### Force 3 — Type B failure jump (discontinuous amplifier)

A Type B failure is a disproved prediction — the system tried to confirm something
and instead found it was wrong. This is the most valuable event the system can
produce. It causes a discontinuous upward jump in boldness — a new frame became
visible — and resets the decay clock:

```
on Type_B_failure:
    boldness(t+1) = boldness(t) × jump_multiplier + jump_base
    jump_multiplier ∈ (1.3, 2.0)  # tunable
    jump_base ∈ (0.1, 0.3)        # tunable
    decay_clock = 0               # reset — system earned the right to be bold
```

The probability of a Type B failure per cycle is:

```
P(Type_B) = base_falsification_rate × boldness(t) × agent_skill_diversity
```

Higher boldness = bolder predictions = more likely to be genuinely falsifiable.
Higher skill diversity = agents are exploring different parts of the space =
more likely one of them finds a real contradiction.

**This is the key nonlinearity.** Boldness increases P(Type_B) which increases
jump frequency which increases boldness. This is the compounding mechanism.
But it is bounded by monoculture pressure (see below).

### The resulting curve

The three forces together produce **punctuated equilibrium**:
- Long periods of local optimization (decay ≈ skill lift, oscillating near stable point)
- Interrupted by Type B jumps (discontinuous upward steps)
- Each jump raises the baseline — the stable point after the jump is higher than before

This is distinguishable from:
- Linear growth (constant slope)
- Pure exponential (no oscillation, no plateaus)
- Convergence (boldness approaches zero)
- Chaos (no stable periods between jumps)

The simulation must produce a recognizable punctuated equilibrium curve to validate
the model. If it produces something else, the model is wrong and should be revised.

## State space

Full system state at time T:

```python
@dataclass
class SystemState:
    t: int                          # cycle number

    # Queue
    queue: List[Item]               # available items
    completed: List[Item]           # shipped items

    # Agents (N parallel)
    agents: List[AgentState]

    # Shared
    shared_skills: Set[str]         # skill library all agents draw from
    anomaly_log: List[Observation]  # observations that don't fit current model
    resolution_log: List[Observation]  # observations the system explained away

    # Vision
    vision_boldness: float          # current system boldness (0.0–1.0)
    predictions: List[Prediction]   # active falsifiable predictions
    calibration_score: float        # historical prediction accuracy

    # Human
    human_engagement_rate: float    # fraction of inflection points accepted
    inflection_points_fired: int    # total inflection points triggered
    inflection_points_declined: int # human declined to engage

@dataclass
class AgentState:
    id: str
    current_item: Optional[Item]
    local_skills: Set[str]          # agent's individual skill set
    boldness: float                 # agent-level boldness
    calibration: float              # this agent's prediction accuracy
    type_b_count: int               # Type B failures produced
    type_a_count: int               # Type A failures (execution wrong)

@dataclass
class Item:
    id: str
    boldness_required: float        # how bold the agent needs to be to attempt
    skill_requirements: Set[str]    # skills needed for high success probability
    generates_skills: Set[str]      # skills produced on success
    prediction: Optional[Prediction]  # the falsifiable claim this item tests

@dataclass
class Prediction:
    id: str
    confidence: float               # how confident the system is (0.0–1.0)
    scope: float                    # how broad the claim is (0.0–1.0)
    falsification_condition: str    # what would prove this wrong
    value: float                    # confidence × scope — the stake

@dataclass
class Observation:
    t: int
    description: str
    fits_model: bool                # True = explained, False = anomaly
    resolution: Optional[str]      # how the system explained it (if fits_model)
```

## Transition rules

One cycle = one time step. All agents act in parallel within a cycle.

### Step 1 — Queue refresh

If queue is empty or below threshold, self-generating vision loop fires:

```python
def generate_next_items(state: SystemState, n_items: int) -> List[Item]:
    # Boldness of generated items scales with system boldness
    # but is also constrained by current skill set
    # Items that require skills the system doesn't have yet
    # are generated with low probability — the system can't
    # see what it can't conceptualize
    base_boldness = state.vision_boldness
    skill_ceiling = len(state.shared_skills) * skill_boldness_coefficient

    for i in range(n_items):
        item_boldness = min(base_boldness, skill_ceiling) * random.gauss(1.0, 0.2)
        yield Item(
            boldness_required=item_boldness,
            skill_requirements=sample_from_adjacent_skills(state.shared_skills),
            generates_skills=project_new_skills(item_boldness),
            prediction=generate_prediction(item_boldness, state.calibration_score)
        )
```

### Step 2 — Item claiming (branch-lock competition)

Each agent attempts to claim an unclaimed item. Agents with higher boldness
claim bolder items. Multiple agents cannot claim the same item (atomic).

```python
def claim_items(agents: List[AgentState], queue: List[Item]):
    random.shuffle(agents)  # randomize claim order
    claimed = set()
    for agent in agents:
        claimable = [i for i in queue
                     if i.id not in claimed
                     and i.boldness_required <= agent.boldness * 1.2]
        if claimable:
            # Agent prefers items near its boldness ceiling
            item = max(claimable,
                      key=lambda i: -abs(i.boldness_required - agent.boldness))
            agent.current_item = item
            claimed.add(item.id)
```

### Step 3 — Execution attempt

Each agent attempts to complete its claimed item:

```python
def attempt_execution(agent: AgentState, item: Item) -> ExecutionResult:
    # Skill match determines success probability
    skill_overlap = len(agent.local_skills & item.skill_requirements)
    skill_gap = len(item.skill_requirements) - skill_overlap
    p_success = 1.0 / (1.0 + exp(-2 * (skill_overlap - skill_gap)))

    roll = random.random()

    if roll < p_success:
        return ExecutionResult.SUCCESS

    # Failure: is this Type A or Type B?
    # Type B: the prediction was bold enough that failing it
    # reveals something real about the system's model
    prediction_value = item.prediction.value if item.prediction else 0
    p_type_b = prediction_value * agent.boldness * falsification_sensitivity

    if random.random() < p_type_b:
        return ExecutionResult.TYPE_B_FAILURE  # disproved prediction — valuable
    else:
        return ExecutionResult.TYPE_A_FAILURE  # execution wrong — rebuild
```

### Step 4 — State updates

```python
def update_state(state, agent, item, result):
    match result:
        case ExecutionResult.SUCCESS:
            # Ship it
            state.completed.append(item)
            state.queue.remove(item)

            # Agent gains skills
            agent.local_skills |= item.generates_skills

            # Shared library gains skills (slower — requires N confirmations)
            state.shared_skills |= {s for s in item.generates_skills
                                    if skill_confirmed_by_multiple_agents(s, state)}

            # Boldness: decay (Force 1) + skill lift (Force 2)
            agent.boldness *= decay_rate
            agent.boldness += len(agent.local_skills) * skill_boldness_coefficient

            # Self-generating vision: propose next item
            state.queue.append(generate_next_item(state, agent))

        case ExecutionResult.TYPE_B_FAILURE:
            # Disproved prediction — most valuable event
            agent.type_b_count += 1

            # Boldness jump (Force 3)
            agent.boldness = agent.boldness * jump_multiplier + jump_base

            # Anomaly log entry
            state.anomaly_log.append(Observation(
                t=state.t,
                description=f"Prediction falsified: {item.prediction.id}",
                fits_model=False
            ))

            # Skills still gained from the attempt
            agent.local_skills |= {s for s in item.generates_skills
                                   if random.random() < 0.5}

            # Calibration update — system was wrong, update belief
            agent.calibration *= 0.95
            state.calibration_score = mean(a.calibration for a in state.agents)

        case ExecutionResult.TYPE_A_FAILURE:
            # Execution wrong — item returns to queue
            agent.type_a_count += 1
            item.attempts += 1
            state.queue.append(item)
            agent.calibration *= 0.98
            agent.boldness *= 0.97  # small confidence hit
```

### Step 5 — Monoculture pressure

Each cycle, agents' local skill sets drift toward the shared library:

```python
def apply_monoculture_pressure(state, monoculture_rate=0.1):
    for agent in state.agents:
        # Skills converge toward shared library
        # This reduces agent diversity over time
        divergent_skills = agent.local_skills - state.shared_skills
        for skill in list(divergent_skills):
            if random.random() < monoculture_rate:
                # Skill gets absorbed into shared library
                state.shared_skills.add(skill)
                # Agent loses the uniqueness of that skill
                # (everyone has it now — it's no longer differentiating)

    # Measure diversity: how different are agents from each other?
    state.skill_diversity = compute_skill_diversity(state.agents)
```

### Step 6 — Human inflection point check

```python
def check_inflection_point(state, config):
    anomaly_density = len(state.anomaly_log) / (state.t + 1)
    calibration_trend = compute_trend(state.calibration_history[-10:])
    velocity = len(state.completed) / (state.t + 1)

    triggered = (
        anomaly_density > config.anomaly_threshold or
        calibration_trend < config.calibration_floor or
        velocity < config.velocity_floor or
        state.vision_boldness < config.boldness_floor
    )

    if triggered:
        state.inflection_points_fired += 1
        # Human reviews: accept (engage) or decline
        if random.random() < state.human_engagement_rate:
            # Human re-enters: injects new boldness, reframes anomalies
            state.vision_boldness = min(1.0, state.vision_boldness * 1.5 + 0.2)
            state.anomaly_log.clear()  # human resolved them
        else:
            state.inflection_points_declined += 1
            # System continues — local maximum risk increases
```

## What the simulation validates (or refutes)

| Hypothesis | Observable signature | Verdict possible? |
|---|---|---|
| Self-generating vision compounds | Boldness grows superlinearly over cycles | Yes |
| Monoculture dominates at high N | Skill diversity collapses, boldness curves converge | Yes |
| Optimal N exists | U-shaped curve: compounding rate vs N | Yes |
| Type B failures are primary driver | Remove Force 3 → boldness plateaus | Yes |
| Human inflection points prevent local maxima | Low engagement_rate → early plateau | Yes |
| Punctuated equilibrium curve shape | Long plateaus + discontinuous jumps | Yes |
| Calibration decay precedes local maximum | Calibration drops before boldness plateaus | Yes |

Each hypothesis is falsified by its observable signature failing to appear.
If none of the signatures appear, the entire model is wrong and should be rebuilt.

## Results (2026-04-17)

Parameters: N=4, 100 cycles, 5 runs, seed=42, human_engagement=0.7

### Force isolation

| Condition | Initial boldness | Final boldness | Curve |
|---|---|---|---|
| Baseline (all 3 forces) | 0.65 | 0.9955 | MONOTONIC_GROWTH |
| Force 1 off (no decay) | 0.66 | 0.9955 | OSCILLATING |
| **Force 2 off (no skill growth)** | **0.61** | **0.2144** | **CONVERGENCE_TO_ZERO** |
| Force 3 off (no Type B jumps) | 0.51 | 0.9985 | MONOTONIC_GROWTH |

**Finding 1 — Skill growth (Force 2) is existential.**
Removing it causes convergence to zero. Removing either other force barely changes the
outcome. The skills library is not enrichment — it is the survival mechanism.

**Finding 2 — Type B failures (Force 3) accelerate early growth only.**
Without Force 3: same final ceiling (0.9985), reached more slowly (initial 0.51 vs 0.65).
Type B failures matter most in cycles 1–40. After that, skill growth dominates.

**Finding 3 — Decay (Force 1) is neutral at 100 cycles.**
Removing it produces identical final boldness. Decay is correctly counteracted by
skill growth in the current parameter regime.

### Human engagement effect

| Engagement rate | Final boldness | Curve |
|---|---|---|
| 0.0 (never) | 0.9955 | MONOTONIC_GROWTH |
| 0.3 | 0.9970 | MONOTONIC_GROWTH |
| 0.7 | 0.9955 | MONOTONIC_GROWTH |
| 1.0 (always) | 0.9940 | OSCILLATING |

**Finding 4 — High human engagement slightly reduces final boldness.**
The system reaches near-identical boldness regardless of engagement rate. At
engagement=1.0 the curve becomes oscillating and the ceiling is marginally lower.
Implication: human re-entry is not a growth driver — it's a recovery mechanism for
genuine stalls. Inflection point thresholds should be set conservatively high.

### N agents comparison

All N values (1, 2, 4, 8, 16) reached final boldness 0.99+. Skill diversity stayed
at 1.0 across all N at 100 cycles. The predicted monoculture dynamic did not appear.

**Finding 5 — The monoculture hypothesis is not testable with the current model.**
Skill diversity stays at 1.0 because new skills are generated faster than monoculture
pressure absorbs them. But real monoculture in otherness is architectural — all agents
share `standalone.md` and reason from the same framework regardless of skill set.
The simulation has no variable for conceptual convergence. This is a model gap.

### Model gaps identified

1. **Architectural monoculture**: all agents share the same reasoning framework
   (`standalone.md`). Skill diversity ≠ conceptual diversity. The model cannot
   distinguish "agents with different skills" from "agents with different worldviews."
2. **External signal injection**: `/otherness.learn` periodically imports foreign
   architectural patterns — this is the only mechanism that breaks conceptual
   monoculture. The simulation has no representation of this.
3. **Ceiling artifact**: all runs converge to ~0.99 boldness because the sigmoid
   success function and skill growth are not properly bounded at high skill counts.
   The model needs a diminishing-returns curve on skill growth.
